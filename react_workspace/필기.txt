클래스 설계시에 생성자에 두개의 매개변수가 전달되어야 한다 

props : 부모 컴포넌트 입장에서 자식컴포넌트한테 값이나 참조를 전달하고자 할때 
        값의 경우는   키="값1"  
        참조의 경우는 키={변수명}, 배열, 객체, 함수
state : 함수나 클래스 내부에서 존재. 상태정보를 저장한다. 
       
클래스의 경우에는 this.state 또는 this.props를 사용해 접근했으나 
함수의 경우에는 this.state 또는 this.props를 못가진다. 대신에 hook을 이용해서 
사용이 가능해졌다. useState와 함수의 매개변수를 이용해 접근가능하다 
그러면서 함수기반 컴포넌트로 흐름이 옮겨온다

#mybatis vs jpa
- 둘 다 쿼리를 작성하는 데 도움을 주는 라이브러리
- mybatis
	- multi-page application 에 적합
	=> database 규모가 크기 때문
	- 찐 쿼리를 작성할 때 쓴다; db 규모가 커서 jpa로는 쿼리를 감당하지 못할 때 mybatis를 주로 쓴다
- jpa
	- single page application에 적합;
	=> db 규모가 작을 때 적합
	- orm 방식

# jsp
- 서버 사이드 템플릿 엔진
- 자바 쪽에서 프론트 템플릿을 작성할 때 사용
- 우리 공부했던 ejs와 완전 유사
- html에서 java 객체를 출력하고 싶을 때 사용
- spring boot에는 쓰지 않는다(boot의 내장 톰캣에서 해석하지 못하기 때문)
=> 그래서 boot에서는 mustache나 thymeleaf를 쓴다

# spa, mpa(multi-page application)
- 모든 서비스에서 spa를 쓰는 게 좋은 것만은 아니다. db 규모가 크거나 하면 mpa가 더 적합할 수도 있다.
- 기존 multi-page로 작성된 것을 spa로 바꾸기에는 너무 고비용 => 최근 새로 개발하는 것에 적용하는 추세
- 무한 스크롤 spa에서만 구현 가능

# SI나 은행 등에서 입사하면 쿼리만 해요의 의미
- join, subquery를 이용해서 데이터 insert, update, delete, select문을 짜는 쿼리를 주로 쓰는 것이다.
- 주로 프로그램이 잘 돌아가는 지 확인 용도 => 해당 쿼리를 돌렸을 때 처리 시간이 오래 걸리면 쿼리를 수정하기도 함
- 데이터 관리 등의 업무는 DBA(database Assistant)가 한다 => 앞에 적은 쿼리 이외의 권한은 주어지지 않을 가능성이 높음

RDBMS 프로그램

oracle, mysql, mssql 등이 있다.
oracle - 대용량 데이터 처리, 색인 순차 검색(인덱스)을 지원한다. 튜닝이라는 작업을 통해서 속도가 더 빠르게 할 수 있다.
오라클과 PL/SQL 프로그래밍 언어 오라클 엔진이 해석하는 언어이다. 반환값이 있으면 함수를 만들고, 반환값이 없으면 프로시저를 만든다.
이 두가지를 합쳐서 함수라고 부르른 언어가 C언어임

대용량 데이터 처리의 경우에는 이걸 웹페이지로 호출하면 100만개정도를 100개단위로 오늘내로 처리해야하는데

PL/SQL로 디비 안에서 프로그램을 만들어서 돌린다. 디비엔진이 스스로 처리 => 말일날 마다 알아서 돌린다.

mysql : 무료버전인 MariaDB를 많이 사용한다. 
장점: 클라우드에서 라이선르를 갖고있는 경우가 많아서 클라우드에 포팅할 때 편함.
단점: 오라클에 비하면 진짜 느리다 -> 검색할 때 너무 느려서 redis라는 메모리 디비를 요즘에 많이 사용하여 속도를 보조하는 편

NOSQL: 비정형 자료 저장

Mongo DB: JSON 객체 배열로 저장, 자바 스크립트를 이용해서 데이터 저장 수정 삭제 검색을 담당
