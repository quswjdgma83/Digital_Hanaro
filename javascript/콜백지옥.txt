DB에서 데이터 처리를 할 때 => 함수가 비동기임
(EX) 상품 구입을 할 경우
    1. 주문 테이블에 주문 정보 INSERT
    2. 쿠폰, 포인트 테이블에 관련 정보 INSERT
    3. 결제 여부가 결제 테이블에 INSERT
    4. 배송 관련 정보가 배송 테이블에 INSERT
    - 1 ~ 4까지 과정이 하나의 트랜잭션 => 한 과정이라도 문제가 생기면 ROLLBACK 해야 함
비동기 처리는 리턴값이 의미가 없음
(ex) let result1 = insertQuery(callBackfn)
=> 이렇게 되어 있으면 callback 함수의 리턴 값이 호출한 함수(insertQuery)에게 전달되는 것이지 result1으로 가는 게 아님
- 콜백 하나가 끝나야 그 다음 콜백이 실행이 됨 => 콜백 안에 콜백을 작성해야 해서 콜백 지옥 발생
